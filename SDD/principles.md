# 專案治理原則 (Project Principles)

本文件定義了 FRM Python 互動式學習平台 PWA 的核心指導原則，確保在開發過程中的一致性與高品質。

## 1. 程式碼品質與標準
- **可讀性優先**：程式碼應清晰易懂，變數命名應具備描述性。
- **模組化設計**：組件應保持專注於單一功能，並盡可能提高重用性。
- **錯誤處理**：所有非同步操作與關鍵邏輯必須包含健壯的錯誤處理機制。
- **文件化**：複雜邏輯應附帶註釋說明，API 與工具函數應具備文件註釋。

## 2. 使用者體驗 (UX)
- **視覺一致性**：遵循現代、簡潔且高級的設計美學，維持整體的配色與佈局協調。
- **響應式支援**：確保在不同螢幕尺寸（手機、平板、桌面）上均能提供極佳的互動體驗。
- **即時回饋**：提供流暢的過渡動畫與操作回饋，減少使用者等待的焦慮感。

## 3. 效能與安全性
- **輕量化載入**：優化資源載入順序，確保 Pyodide 與核心元件能夠快速呈現。
- **記憶體管理**：有效釋放 Python 執行過程中的資源，防止瀏覽器記憶體洩漏。
- **離線支援**：充分利用 PWA 特性，確保在無網路環境下亦能存取核心學習內容。

## 4. 決策指引
- 當技術方案衝突時，應以「使用者學習體驗」為首要考量。
- 優先採用標準化的 Web 技術與穩定的開源套件。
